saltchan FAQ

How does this work?

For every thread you visit or create, we'll generate a unique keypair on the fly that we use to sign and optionally encrypt your submission. This is powered by a JavaScript port of tweetnacl.

Is this secure?

Maybe. We can never be certain. Not unless a third party has conducted a thorough review of the underlying codebase.

Feel free to browse the source and assure yourself that the server truly does not know anything about an encrypted submission.

Any caveats?

For now, submitting an encrypted image will not encrypt the filename, which will stay publicly visible.

If you'd like to encrypt an animated GIF, only the first frame will be used.
